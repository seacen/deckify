04 / 09 Strategy  ·  Type B
Constraints vs. Freedom

A toolkit, not a menu.

The DS defines hard constraints (locked) and reusable components (reach-for) — but encourages bespoke compositions built from system tokens.

Constraints  ·  locked

What you never break

  • §2 colour tokens only — no ad-hoc hex
  • Montserrat only — no serif fonts
  • 12 px readability floor
  • Logo on every slide
Bespoke  ·  encouraged

What to invent freely

  • Quartered-circle Annual Report motifs
  • Product mosaics on `--primary` ground
  • Hero numerals with `--spark` endcap
  • Tokens + Montserrat + ≥ 12 px = in-system

07 / 09 Reveal  ·  Type D
Hover or tap to reveal  ·  click toggles on mobile

Two stops on the same blue ladder.

Front: --primary

P&G blue with the sphere monogram. Used on every cover and dark conclusion bar.

Reveal back  →
Back  ·  --primary-deep

Deep navy — the second stop.

A darker navy for flip-card backs. Back and front read as two stops on one ladder — never `--accent` cyan on `--primary` (the blue-on-navy trap).

Front: hover or tap

Desktop hover triggers rotateY(180°). Mobile `onclick` toggles `.on`.

Reveal back  →
Back  ·  mobile fix

CSS :hover doesn’t exist on touch.

Mobile kills 3D transforms and uses display:none / display:flex to swap faces. `classList.toggle('on')` is the bridge.
