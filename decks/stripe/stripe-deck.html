08 · The Stack, in Order
Each layer compounds against the one before it
The Stripe stack didn't ship as one release. It accreted — Payments first, then everything that depends on the same primitives. The order is the moat.
Layer 01 · Core
Payments — accept cards, wallets, bank debits
The original API — seven lines of code, one integration. Everything else hangs off the same primitive.
Layer 02 · Platforms
Connect — marketplaces and platforms
Onboard third-party sellers, route funds, manage payouts. Same API surface; different operator question.
Layer 03 · Recurring
Billing — subscriptions and usage-based pricing
Recurring revenue infrastructure — invoices, dunning, prorations, usage metering. Compounds on top of Payments.
Layer 04 · Now
Atlas, Climate, Sigma, Issuing — the whole stack
Incorporation, carbon removal, SQL on Stripe data, card issuing — every layer reuses the API surface and the Dashboard.
